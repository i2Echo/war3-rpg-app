## 2. 命名规范

### 2.1 命名统一规则

**禁止使用的符号**：
- 装备名称中**不得**包含 `.`（点号）或 `·`（间隔号）
- 所有已存在的带这些符号的名称应统一去除

**标准命名格式**：
- 基础名称：直接使用汉字或字母，无需分隔符
- 示例：`幻剑天陇`（非 `幻剑.天珑` 或 `幻剑·天珑`）

### 2.2 名称归一表（同物异名统一）

以下写法统一为右侧标准名：

- `玛利亚圣石` → `玛丽亚圣石`
- `摩西戒杖` → `摩西诫杖`
- `酸雨羽扇` → `酸与羽扇`
- `枯叶` → `枯夜`
- `巽狼刀` → `撰狼刀`
- `幻剑.天珑` / `幻剑·天珑`/`幻剑天陇` → `幻剑天珑`
- `斯贝斯之诅咒` → `斯贝斯的诅咒`
- `罂栗骨杖` → `罂粟骨杖`
- `磨枪诺亚` → `魔枪诺亚`
- `真.饕餮戒指` / `真·饕餮戒指` / `真饕餮扳指` / `真.饕餮扳指` → `真饕餮扳指`
- `天狼斜眼` → `天狼邪眼`
- `海洋之星` → `海洋之心`

## 3. 装备特殊属性标记规则 ⭐新增⭐

### 3.1 特殊属性类型及含义

装备可以具有以下三种特殊属性，用于标识特殊获取方式或限制。**注意：这些标记是装备的属性，不是名称的一部分。**

#### **[唯一]** 属性
- **含义**：该装备为全图任务唯一装备，无法通过炼化公式获得
- **获取方式**：仅通过特定任务/剧情获得
- **炼化规则影响**：
  - 该装备**不能作为炼化产物**（除非在专属炼化中明确指定）
  - **涉及该装备的随机公式将被废除**（不再生效）
  - 该装备**可作为炼化材料**，需在公式中明确指定
- **原始数据标记**：`生锈的发簪(唯一)`、`焚寂(唯一)`、`漆黑之牙(唯一)`
- **存储名称**：`生锈的发簪`、`焚寂`、`漆黑之牙`（不含括号后缀）
- **显示格式**：带右下标，如 `焚寂₍唯一₎`、`漆黑之牙₍唯一₎`

#### **[卷轴合成]** 属性
- **含义**：该装备只能通过特定的卷轴合成获得，无法通过常规炼化公式获得
- **获取方式**：使用特定卷轴进行合成
- **炼化规则影响**：
  - 涉及此类装备的公式，**标题应改为"卷轴合成公式"**而非"炼化公式"
  - **涉及该装备的随机公式将被废除**（不再生效）
  - 该装备**可作为炼化材料**
- **原始数据标记**：`魔导师护胸改(卷轴合成)`、`云霞舞梦改(卷轴合成)`、`昆仑仙履(卷轴合成)`
- **存储名称**：`魔导师护胸改`、`云霞舞梦改`、`昆仑仙履`
- **显示格式**：带右下标，如 `昆仑仙履₍卷轴₎`
- **公式示例格式**：`卷轴合成公式：XX卷轴+YY=ZZ，100%`

#### **[一次]** 属性
- **含义**：该装备只能炼化一次，之后无法再次使用相同公式进行炼化
- **获取方式**：通过特殊炼化公式获得
- **炼化规则影响**：
  - 该炼化属于**特殊炼化**，通常成功率为100%
  - **涉及该装备的随机公式将被废除**（不再生效）
  - 该装备**可作为炼化材料**
  - 同一存档中，该公式只能使用一次
- **原始数据标记**：`机械血珠(一次)`、`蓬莱仙珠(一次)`、`党魂劫焰(一次)`
- **存储名称**：`机械血珠`、`蓬莱仙珠`、`党魂劫焰`
- **显示格式**：带右下标，如 `蓬莱仙珠₍一次₎`
- **公式示例**：`雾幻云珠+仙气葫芦=蓬莱仙珠，100%，只能炼化一次`

### 3.2 特殊装备在随机公式中的处理 ⭐新增⭐

**核心原则**：特殊装备（带三种属性标记之一的装备）**不作为随机公式的炼化产物**。

#### 规则说明：

1. **随机公式废除规则**：
   - 如果一个随机公式的产物中包含特殊装备，该随机公式**整条废除**
   - 例：`公式A = 随机S战武` 如果随机S战武池中包含 `界罗杀生[唯一]`，则该公式废除
   
2. **明确公式保留规则**：
   - 明确指定特殊装备为产物的公式**保留**
   - 例：`焚寂+米迦勒=废墟之刃，100%` 保留（专属炼化）
   
3. **材料使用规则**：
   - 特殊装备**可以作为炼化材料**
   - 例：`焚寂+XX=YY` 合法（焚寂作为材料）

#### 实施方案：

**方案A：废除含特殊装备的随机公式**
```
原公式：B战武+A力量=阿修罗，33%；幻魂胧月，33%；焚寂，33%

处理：
- 如果 焚寂 标记为 [唯一]
- 则该公式中 焚寂 的产出路径废除
- 修正为：B战武+A力量=阿修罗，50%；幻魂胧月，50%
```

**方案B：随机池自动过滤**
```
原公式：S战武+AB任意=随机S力量战武，100%

S力量战武列表：
- 党魂劫焰[一次] ❌ 排除
- 魔枪诺亚 ✅ 保留
- 轩辕剑 ✅ 保留
- 真漆黑之牙[一次] ❌ 排除
- 界罗杀生[唯一] ❌ 排除
- ...

随机S力量战武池 = 仅包含无特殊属性的装备
```

**采用方案**：**方案B**（自动过滤更易维护）

### 3.3 特殊属性标记的数据处理 ⭐新增⭐

#### 数据存储格式：

**原始数据**（`西方世界的劫难3炼化公式.md`）：
```markdown
#### A力量：
- **武器类**：幻魂胧月，死亡之蝎(唯一)，洪荒戟，焚寂(唯一)
```

**解析后存储**：
```javascript
equipmentDatabase = {
  items: [
    { name: "幻魂胧月", grade: "A", category: "战武", attribute: "力量" },
    { name: "死亡之蝎", grade: "A", category: "战武", attribute: "力量", special: "唯一" },
    { name: "洪荒戟", grade: "A", category: "战武", attribute: "力量" },
    { name: "焚寂", grade: "A", category: "战武", attribute: "力量", special: "唯一" }
  ]
}
```

**解析规则**：
1. 读取原始数据时，检测装备名称是否包含 `(唯一)`、`(卷轴合成)`、`(一次)`
2. 提取纯净名称（去除括号和后缀）
3. 将后缀转换为 `special` 属性字段：
   - `(唯一)` → `special: "唯一"`
   - `(卷轴合成)` → `special: "卷轴合成"`
   - `(一次)` → `special: "一次"`
4. 无后缀的装备，`special` 字段为 `undefined` 或不设置

#### UI显示格式：

```html

<span class="item-name">魔枪诺亚</span>

<!-- 特殊装备（带右下标） -->
<span class="item-name">
  焚寂<sub class="item-special-mark">唯一</sub>
</span>

<!-- 或使用简化标记 -->
<span class="item-name">
  蓬莱仙珠<sub class="item-special-mark">一次</sub>
</span>
```

## 4. 公式语义与顺序规则

1. 非固定炼化：顺序敏感，必须按公式左右位置处理。
2. 固定炼化：仅"明确标注为固定炼化/无视顺序"的公式可无视顺序。
3. 排除条件（`非` / `不包括`）：按"备注限制"处理，不单独转为硬性过滤表达式。

## 5. 概率规则

1. 单条公式多百分比时，按该条描述中的最终值解释。
2. `33%/66%`、`33%+33%+33%` 表示概率分档（如 1/3、2/3），不按"合计=99%错误"处理。
3. 若展开项未单独给概率，默认与同类展开项一致。
   - 例：`B鞋+CD任意=随机B鞋` 使用同类 `B鞋` 互转概率口径。
4. "超雄铁骑 20%"仅作该物品的文字备注，不改变通用 `S鞋` 互转公式概率体系。

## 6. 解析约定

1. 支持多品级缩写自动展开：`AB任意`、`CD任意` 等。
2. 纯品级条件统一写作 `X任意`（如 `A任意`）。
3. 缺失必要结构（如缺少 `=`）的行默认不入公式库，除非在本标准补充为明确公式。

## 7. 已补充固化公式

- `光络镜杖+光羽之靴=雅典娜神杖`（100%，固定炼化，无视顺序）

## 8. 随机池边界

- `随机A饰品，不包括东正护符...` 仅作用于"饰品+任意篇"的随机A饰。
- 不自动外扩到其他随机A饰规则。
- **新增**：所有随机池**自动排除**具有特殊属性标记（[唯一]、[卷轴合成]、[一次]）的装备。

**随机池构建优先级**：
1. 首先按品级、部位、属性筛选基础池
2. 排除特殊属性装备（[唯一]、[卷轴合成]、[一次]）
3. 再按公式中的明确排除条件过滤
4. 最终得到可用随机池

## 9. 展示与查询口径

1. 主结果去重按"第一步主炼化公式"去重。
2. 同主公式仅展示一条，后续由用户点击继续钻入查询。
3. 公式展示连接符使用 `=`。
4. **新增**：装备名称显示时，特殊装备需添加右下标标记：
   - 普通装备：`魔枪诺亚`
   - 特殊装备：`焚寂₍唯一₎`、`蓬莱仙珠₍一次₎`、`昆仑仙履₍卷轴₎`
   - HTML实现：`<span>焚寂<sub>唯一</sub></span>`

## 10. 后续变更流程（建议）

当 `md` 有新增/修订时，按以下流程维护：

1. 先更新 `md` 原文。
2. 在本标准追加"变更条款"（不要直接覆盖旧条款）。
3. 若涉及同物异名，先补"名称归一"再改公式。
4. **新增**：若需标记特殊装备，在装备名称后添加 `(唯一)`、`(卷轴合成)` 或 `(一次)` 后缀。
5. **新增**：更新后需重新解析装备数据库，提取特殊属性标记。
6. 跑一次构建与类型校验，确认可用。

### 装备数据解析示例

```javascript
// 解析装备列表，提取特殊属性
function parseEquipmentList(rawText) {
  const items = [];
  const itemPattern = /([^，,]+?)(\(唯一\)|\(卷轴合成\)|\(一次\))?(?=[，,]|$)/g;
  
  let match;
  while ((match = itemPattern.exec(rawText)) !== null) {
    const name = match[1].trim();
    const suffix = match[2];
    
    const item = { name };
    
    if (suffix === '(唯一)') {
      item.special = '唯一';
    } else if (suffix === '(卷轴合成)') {
      item.special = '卷轴合成';
    } else if (suffix === '(一次)') {
      item.special = '一次';
    }
    
    items.push(item);
  }
  
  return items;
}

// 示例：
// 输入："幻魂胧月，死亡之蝎(唯一)，洪荒戟，焚寂(唯一)"
// 输出：
// [
//   { name: "幻魂胧月" },
//   { name: "死亡之蝎", special: "唯一" },
//   { name: "洪荒戟" },
//   { name: "焚寂", special: "唯一" }
// ]
```

### 随机池过滤示例

```javascript
// 构建随机池，自动排除特殊装备
function buildRandomPool(items) {
  return items.filter(item => !item.special);
}

// 示例：
const allAWarriorWeapons = [
  { name: "幻魂胧月", grade: "A", category: "战武", attribute: "力量" },
  { name: "死亡之蝎", grade: "A", category: "战武", attribute: "力量", special: "唯一" },
  { name: "洪荒戟", grade: "A", category: "战武", attribute: "力量" },
  { name: "焚寂", grade: "A", category: "战武", attribute: "力量", special: "唯一" }
];

const randomPool = buildRandomPool(allAWarriorWeapons);
// 结果：[{ name: "幻魂胧月", ... }, { name: "洪荒戟", ... }]
```
